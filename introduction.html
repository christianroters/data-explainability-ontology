Ontology for Data Explainability

Abstract

This ontology models the core concepts and relationships within the domain of data explainability. Its primary goal is to provide an overview of the methods, characterizations, and governance principles that contribute to making data more understandable and trustworthy.

Motivation

The motivation for this ontology stems from the need to make every step in the ML/AI pipeline transparent and explainable. Data explainability slots into the pre-modeling phase, enabling stakeholders to already understand the data before it is used. This can make models more performant, fairer and trustworthy, by for example identifying and mitigating biases in the data beforehand, ensuring privacy and making the data more transparent. The ontology structures all the relevant concepts and methodologies in this domain and providing a comprehensive overview on the relationships between them. This can enable researchers, practitioners and stakeholders to better understand the field of data explainability, identify gaps and opportunities for improvement, and ultimately build more trustworthy AI systems.

Core Concepts

The ontology is structured around three main pillars that contribute to the central goal of Data Explainability:

  • Methodology: The processes and techniques applied to data (e.g., ExploratoryDataAnalysis, FeatureEngineering).
  • Characterization: The artifacts and findings that describe the data (e.g., Biases, Anomalies, StatisticalCharacterization).
  • Governance: The principles that guide the process (e.g., Fairness, Privacy).

Additionally, on this base level of concepts, trust and transparency can be found, as they are complimentary to data explainability.

Key Semantic Patterns: A Walkthrough

Example 1: The Lifecycle of a Bias

The ontology captures the lifecycle of a finding like a bias. First, a process from the Methodology class identifies it using the produces property (or its sub-property identifies). Later, a different process can address it using the mitigates property. This leads to methodology improving the data's explainability, modeled by the contributesTo property, which in turn contributes to building Trust. The diagram below illustrates this flow.

Example 2: Method Work Flow

The ontology also models the sequence of methods applied to data, which methods precede one another and what they produce. Additionally it is also modeled that Governance(Step: 1) aspects influence these methods. For instance, FeatureEngineering(Step: 2.3) is typically preceded by ExploratoryDataAnalysis(Step: 2.1) or DataManipulation(Step: 2.2). This is represented using the isPrecededBy property.
